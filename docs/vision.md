# 愿景与规划

## 我们在做什么

理工科研究生的日常科研中，大量时间消耗在重复性的信息获取、整理和格式化上——文献检索、数据采集、报告撰写、格式调整。这些工作不创造知识价值，但占据大量精力。

Hermes Research Station 的目标是用 AI Agent 替代这些重复劳动，让研究者专注于真正的科研思考。

## 三阶段路线

### 阶段一：数据采集基础设施（当前）

**目标**：让 Agent 能可靠地从互联网获取科研所需的一切信息。

核心能力：
- 多层网页抓取，自动应对反爬机制
- 学术文献检索（跨平台聚合、元数据提取）
- 结构化数据提取与质量评估
- 长时间稳定运行（WSL2 环境优化）

已完成：
- L0-L3 四层抓取架构，含自动降级与重试
- Edge CDP 真实浏览器层（绕过大部分反爬）
- 认知插件 16 子系统全部激活
- 子 Agent 编排通过压力测试

### 阶段二：智能科研助手

**目标**：从"能采集数据"进化到"能辅助思考"。

规划能力：
- 自然语言驱动的文献调研：给定研究方向 → 自动检索 → 生成结构化综述
- 跨终端交互：CLI 深度工作、即时通讯快速查询、移动端随时响应
- 探索新形态终端（AI 眼镜等）在学术会议、实验现场的实时辅助场景
- 知识库积累：每次调研的结果沉淀为可检索的个人知识库

### 阶段三：可复现的科研工具链

**目标**：将个人工具打磨为可分享的开源项目。

规划方向：
- 工作流模板化：文献调研、数据分析、实验记录等标准化流程
- 开箱即用的部署方案：一键克隆 + 配置即用
- 面向 AI 时代的基础科研设施：持续迭代，适配新模型、新终端、新科研范式

## 设计原则

1. **不造已有轮子** — Agent 框架用 Hermes，模型用现成 API，专注应用层
2. **先验证后实现** — PoC 通过再做完整实现，数值证明优于美观
3. **可迁移** — 配置模板化，一键克隆到新机器即可用
4. **渐进式** — 每个阶段独立可用，不依赖后续阶段
